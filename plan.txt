🎯 프로젝트 개요
나는 Firebase를 사용해 '아키네이터(akinator)'처럼 사람을 추리하는 웹앱을 만들고 싶다.
대상은 학교 친구 약 100명 정도이며, 사용자가 "예/아니오"로 대답하면서 후보를 좁혀 최종 인물을 맞히는 구조다.
html, css, javaskript와 Firebase Firestore를 사용하고, 관리자만 인물 등록/수정 가능하게 구현할 계획이다.

---

📦 핵심 요구사항
1. 관리자용 인물 등록 페이지
   - 이름, 성별, 학년, 동아리, 안경 착용 여부, 키 등 객관적 속성 선택
   - 이름 초성 포함 여부, 성, 이름 끝 글자 등 문자열 기반 필터링 속성 생성
2. 게임 페이지
   - 질문 순서대로 사용자에게 제시
   - 객관적 질문 위주: 예/아니오/모르겠습니다
   - 질문 답변 기반 후보 필터링
   - 후보가 1명 남으면 결과 출력, 0명 남으면 "오류" 표시
   - 필요한 질문만 출력 (모든 질문을 물어보는게 아니라)
3. 필터링 로직
   - Firestore에서 쿼리 기반 후보 필터링
   - Boolean 및 문자열 속성 모두 지원
4. 더 필요한 것
   - 질문 자동 최적화
   - 통계 페이지 (이 인물을 ~명이 찾아봤습니다)
   - 추가 속성, 확장 질문
6. 모든 페이지 분리
메인 페이지, 게임 페이지, 결과 페이지, 관리자 페이지 분리 (예: index.html, play.html, result.html, admin.html)
7. 디자인
메인 페이지를 아키네이터와 완전히 똑같게 만든다. 전체적인 사이트 디자인도 아키네이터와 똑같이 만든다.
